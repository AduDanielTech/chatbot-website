About Project

The project is a chatbot developed using Dialogflow and Django. The chatbot can answer all Javascript questions and can be accessed only by authenticated users. The users need to log in or register using the authentication page, and only after successful authentication, they can access the chatbot. The chatbot is integrated with Django using the Django REST framework as the frontend which requires html5, css and javascript and uses PostgreSQL as the database to store user data. The project can be further enhanced by adding more functionalities to the chatbot and integrating it with other technologies.

Setting up Django

  • Open your command prompt or terminal and navigate to your preferred directory where you want to create your project.
  • Create a new virtual environment by running the following command: `python -m venv env`
  • Activate the virtual environment by running the following command: `source env/bin/activate` (for Mac/Linux) or `env\\Scripts\\activate` (for Windows).
  • Install Django by running the following command: `pip install django`.
  • Create a new Django project by running the following command: `django-admin startproject {project_name}`.
  • Navigate into the newly created project directory by running the following command: `cd {project_name}`.
  • Create a new Django app by running the following command: `python manage.py startapp {app_name}`.

  • Create HTML Templates

    --Create a directory called "templates" in your Django app directory (if it doesn't exist).
    --Inside the "templates" directory, create an HTML file for each page or component you want to include.
    --For example, you can create a file called "home.html" for the home page.
    --Place your HTML code inside these template files.

  • Define URL Patterns

    --Open the "urls.py" file in your Django app directory.
    --Import the views module that contains your view functions.
    --Add a URL pattern that maps a URL path to a view function.
    --Specify the name of the HTML template to render for that URL pattern.
    --For example, you can add a URL pattern like this: path('', views.home, name='home').

  • Create View Functions

    --Open the "views.py" file in your Django app directory.
    --Define a view function for each URL pattern you added.
    --Each view function takes a request parameter and returns an HttpResponse object.
    --Inside the view function, use the render function to render the HTML template.
    --For example, a view function for the home page could look like this:
         from django.shortcuts import render
              def home(request):
                 return render(request, 'home.html')

  • Link CSS Files

    --Create a directory called "static" in your Django app directory (if it doesn't exist).
    --Inside the "static" directory, create another directory with the same name as your app.
    --Place your CSS files inside this app-specific "static" directory.
    --In your HTML templates, include the CSS files using the static template tag.
    --For example, you can include a CSS file like this: <'link' rel="stylesheet" href="-static 'appname/style.css' -">.
    --Make sure to load the static template tag at the beginning of your template file: load static .

  • Collect Static Files (for production)

    --If you are running your Django project in production mode, you need to collect static files.
    --Run the command python manage.py collectstatic in your project's root directory.
    --This command will gather all static files from your apps and copy them to a single location.

  • Start the Django Development Server

    --Run the command python manage.py runserver in your project's root directory.
    --Open a web browser and visit the URL corresponding to your Django app and the defined URL pattern.
    --You should see the rendered HTML with the linked CSS styles.

Integrating Django with PostgreSQL

    To connect Django to PostgreSQL, follow these steps:

  • Install the PostgreSQL database server and client on your system.
  • Create a new database in PostgreSQL using the psql command-line interface or a GUI tool like pgAdmin.
  • Install the psycopg2 package for Python, which is the PostgreSQL adapter for Python.
  • Update your Django project's [settings.py](http://settings.py/) file to include the database configuration settings. Specifically, you'll need to update the DATABASES dictionary with the appropriate values for the NAME, USER, PASSWORD, HOST, and PORT keys. For example:
    ```
    DATABASES = {
      'default': {
         'ENGINE': 'django.db.backends.postgresql',
          'NAME': 'mydatabase',
          'USER': 'mydatabaseuser',
          'PASSWORD': 'mypassword',
          'HOST': 'localhost',
          'PORT': '5432',
      }
     }
    ```
  • Run the Django migrations to create the necessary database tables and schema using the command python manage.py migrate.
  • Test the database connection by running some database queries using Django's built-in ORM or a database client like pgAdmin.

    • By following these steps, you can connect Django to PostgreSQL and start using the power of a relational database in your Django project.

Rundown of the Views.py File

  • register(request)

    This function handles the registration process. It checks if the request method is POST and retrieves the submitted form data such as first name, last name, username, email, password, and confirm password. It performs various validations, such as checking if the password meets the specified criteria and if the username or email already exists. If all validations pass, a new User object is created using the `create_user` method, and a UserProfile object is also created. The UserProfile is associated with the User and stores additional information like email verification status. Finally, the user is redirected to the login page.

  • login(request)

    This function handles the user login process. It checks if the request method is POST and retrieves the submitted username and password. It uses the authenticate function from Django's authentication system to verify the credentials. If the user is authenticated and their email has been verified (based on the UserProfile), they are logged in using the login method. Otherwise, they are redirected to the registration verification page or shown an error message.

  • logout(request)

    This function logs out the currently authenticated user using the logout method from Django's authentication system. After logging out, the user is redirected to the homepage.

  • registered(request)

    This function handles the account verification process after registration. If the request method is POST, it retrieves the entered OTP (One-Time Password) and compares it with the generated OTP using the `generateOtp` method from the generator object. If the OTPs match, the UserProfile associated with the user is retrieved, and the email_verify field is set to True to indicate successful verification. The changes are saved, and the user is redirected to the homepage. If the OTP is invalid, an error message is displayed.

  • registereder(request)

    This function is used for requesting and generating an OTP during the registration verification process. If the request method is POST, an OTP is generated using the `generate` method from the `generator` object. The OTP is then sent via email (implementation not shown). Finally, the user is redirected to the registered view. If the request method is not POST, it displays any error message stored in the storeUser.Error variable.

    • It's important to note that the code assumes the existence of related models, such as `UserProfile`, and imports the necessary modules and classes. The `storeUser` object is used to store temporary data related to the registration process.

Setting up Dailogflow

    To set up Dialogflow, follow these steps:
  • Create a new Dialogflow agent by logging in to the Dialogflow console and clicking on the 'Create Agent' button. Give your agent a name and select the default language and time zone for your agent.
  • Enable the Knowledge Connectors feature in your agent settings. This will allow you to connect your agent to external knowledge sources like a Knowledge Graph API.
  • Create an intent in your agent that handles user queries related to your knowledge base. You can define training phrases and responses for this intent to help your agent understand and respond to user queries.
  • Configure the Knowledge Connectors feature to connect your agent to your knowledge source (e.g., Knowledge Graph API). You'll need to provide authentication credentials and a query to retrieve information from your knowledge source.
  • Test your agent by sending queries to it and evaluating the responses. You can use the Dialogflow console or a client library to communicate with your agent programmatically.

    • By following these steps, you can set up a Dialogflow agent that leverages external knowledge sources to provide intelligent responses to user queries.

Google Knowledge Search API File

    This code defines a Django view that handles two functionalities:

  • index view

    - This view checks if the user is authenticated. If the user is authenticated, it renders the 'index.html' template. If not, it redirects the user to the home page.
    - It is assumed that the 'index.html' template contains the appropriate content to be displayed when the user is authenticated.

  • process_input view

    - This view handles a POST request that contains an 'inputData' parameter. It performs a search using the Google Knowledge Graph Search API to retrieve information related to the `input data`.
    - The input data is used as a query parameter for the API request. The response is processed to extract relevant information.
    - If successful results are obtained, the detailed description of the top three search results is extracted and concatenated into the 'results' string. This string is then returned as a JSON response.
    - If no results are found or if there is an error, an appropriate error message is returned as a JSON response.
    - If the request method is not POST or there is an error, the view redirects the user to the home page with an HTTP response indicating a bad request.

    • The code imports necessary modules and libraries (`render`, `redirect`, `JsonResponse`, `HttpResponseBadRequest`, `requests`, and `TextBlob` ). It also sets up the API URL and API key for the Google Knowledge Graph Search API.
